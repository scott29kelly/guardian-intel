6. Exported `useCustomerArtifacts(customerId: string | undefined)` function:

   a. `useQueryClient()` for cache invalidation.

   b. `statusQuery` via `useQuery`:
      - `queryKey: artifactKeys.customer(customerId!)`
      - `queryFn`: fetches `GET /api/decks/status/${customerId}`, throws on non-ok response with `new Error("Failed to fetch artifact status")`, returns `response.json()` typed as `ArtifactsResponse`.
      - `enabled: !!customerId`
      - `refetchInterval` callback (per D-13/D-14 -- poll while processing):
        - If `!data?.hasDeck` return `false` (no deck row exists, no polling needed)

      - `staleTime: 2000`

      return {

        isAllTerminal: statusQuery.data?.artifacts

        isAnyProcessing: statusQuery.data?.artifacts
          ? Object.values(statusQuery.data.artifacts).some(

            )
          : false,
        // Actions
        generate: generateMutation.mutateAsync,
        isGenerating: generateMutation.isPending,

      };

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | useCustomerArtifacts fetch | accept | Backend enforces assertCustomerAccess on both endpoints (Phase 7 D-04). Hook inherits session cookie. No client-side auth bypass possible. |
| T-09-02 | Information Disclosure | artifactKeys query key | accept | Query keys contain customerId which is already visible in the URL. No secrets in query keys. |
| T-09-03 | Denial of Service | refetchInterval polling | mitigate | refetchInterval callback returns false when all artifacts are terminal or hasDeck is false, preventing infinite polling. 3s interval is bounded. |
</threat_model>
