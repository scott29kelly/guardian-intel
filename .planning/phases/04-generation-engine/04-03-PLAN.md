    **File 1: `src/app/api/ai/generate-infographic/batch/route.ts`**

    Create POST handler for batch generation:
    1. Same auth pattern
    2. Import `generateInfographic` from infographicGenerator, `cacheResult` from infographicCache
    3. Import `getPresetsForBatch` from templates index
    4. Parse body as `BatchRequest`: validate `customerIds` (string[], non-empty), `autoSelectPresets` (boolean)
    5. Generate a `jobId` using `crypto.randomUUID()`
    6. **In-memory job store:** Create a module-level `Map<string, BatchJob>` where:
       ```typescript
       interface BatchJobCustomer {
         customerId: string;
         status: 'pending' | 'generating' | 'complete' | 'error';
         result?: InfographicResponse;
         error?: string;
       }
       interface BatchJob {
         jobId: string;
         customers: BatchJobCustomer[];
         startedAt: Date;
         completedAt?: Date;
       }
       ```
    7. Initialize the job with all customers as 'pending'
    8. **Fire and forget:** Start generation in background (do NOT await). Use an async IIFE:
       ```typescript
       (async () => {
         const batchPresets = getPresetsForBatch();
         const batchPresetId = batchPresets[0]?.id || 'pre-knock-briefing';
         for (const entry of job.customers) {
           entry.status = 'generating';
           try {
             const response = await generateInfographic({
               customerId: entry.customerId,
               mode: 'preset',
               presetId: batchPresetId,
               audience: 'internal',
             });
             entry.status = 'complete';
             entry.result = response;
             // Cache each result
             await cacheResult({ customerId: entry.customerId, presetId: batchPresetId, imageData: response.imageData, generatedAt: new Date(), expiresAt: new Date(Date.now() + 86400000), modelStrategy: response.model }, 'internal');
           } catch (err) {
             entry.status = 'error';
             entry.error = err instanceof Error ? err.message : 'Unknown error';
           }
         }
         job.completedAt = new Date();
       })();
       ```
    9. Return `NextResponse.json({ jobId, customerCount: customerIds.length })` immediately with status 202

    **File 2: `src/app/api/ai/generate-infographic/batch/[jobId]/route.ts`**

    Create GET handler for batch status:
    1. Same auth pattern
    2. Import the `batchJobs` Map from the batch route file (export it from batch/route.ts)
    3. Extract `jobId` from `params.jobId` (Next.js dynamic route param)
    4. Look up job from Map. If not found, return 404 `{ error: 'Batch job not found' }`
    5. Compute summary stats: `total`, `pending`, `generating`, `complete`, `error` counts
    6. Return:
       ```json
       {
         "jobId": "...",
         "status": "in_progress" | "complete",
         "total": N,
         "completed": N,
         "failed": N,
         "customers": [{ customerId, status, result?, error? }]
       }
       ```
    7. Status is "complete" when all customers are either 'complete' or 'error'

    Note: In production, the batch job store should use Redis. For v1, the in-memory Map is sufficient since batch jobs are short-lived and the API runs on a single server instance.
