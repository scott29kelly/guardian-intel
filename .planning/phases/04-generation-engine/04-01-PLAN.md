    Create `src/features/infographic-generator/services/infographicCache.ts` that wraps the existing `src/lib/cache.ts` utilities:

    1. **Constants:**
       - `INFOGRAPHIC_CACHE_NAMESPACE = "guardian:cache:infographic"` (follows pattern from CACHE_NAMESPACES in cache.ts)
       - `INFOGRAPHIC_CACHE_TTL = { standard: 86400, leaveBehinds: 604800 }` (24hr = 86400s, 7 days = 604800s)

    2. **Key format:** `infographic:{customerId}:{presetId}:{YYYY-MM-DD}` -- build using string template, not buildCacheKey (since infographic is not in CACHE_NAMESPACES)

    3. **`getCached(customerId: string, presetId: string): Promise<InfographicCacheEntry | null>`**
       - Build key with today's date: `${INFOGRAPHIC_CACHE_NAMESPACE}:${customerId}:${presetId}:${new Date().toISOString().slice(0,10)}`
       - Call `cacheGet<InfographicCacheEntry>(key)`
       - Return the entry or null

    4. **`cacheResult(entry: InfographicCacheEntry, audience: InfographicAudience): Promise<boolean>`**
       - Determine TTL: `audience === 'customer-facing' ? INFOGRAPHIC_CACHE_TTL.leaveBehinds : INFOGRAPHIC_CACHE_TTL.standard`
       - Build key from entry.customerId + entry.presetId + entry.generatedAt date
       - Call `cacheSet(key, entry, ttl)`
       - Return success boolean

    5. **`invalidateForCustomer(customerId: string): Promise<boolean>`**
       - Import `Redis` from `@upstash/redis` and check env vars for availability
       - If Redis available: use `redis.scan` with match pattern `${INFOGRAPHIC_CACHE_NAMESPACE}:${customerId}:*` then `redis.del` all matched keys (same scan-and-delete pattern as `cacheInvalidateNamespace` in cache.ts)
       - If Redis not available (in-memory fallback): iterate over keys -- since we don't have access to the in-memory map from cache.ts, store a local Set<string> of cached keys and delete matching ones
       - Return success boolean

    6. **Export** all three functions and the TTL constants.

    Do NOT add "infographic" to the CACHE_NAMESPACES in cache.ts -- keep it self-contained in this file. Do NOT implement service worker caching (that's Phase 6 INFOG-027).
