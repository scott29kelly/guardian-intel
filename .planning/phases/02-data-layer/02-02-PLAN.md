    Create `src/features/infographic-generator/services/promptComposer.ts`:

    1. **Imports:** TopicModule, ModelCapabilities, InfographicAudience, ScoringResult from infographic.types. getBrandingForInfographic from brandingAssets. BrandingConfig from deck-generator types.

    2. **`buildBrandingInstructions(branding: BrandingConfig, audience: InfographicAudience): string`** -- returns a prompt section with:
       - Color palette: "Primary: ${branding.colors.primary}, Secondary: ${branding.colors.secondary}, Accent: ${branding.colors.accent}, Background: ${branding.colors.background}"
       - Font instructions: "Use clean sans-serif typography (Inter family)"
       - Logo placement: "Place company logo in top-left corner" (internal) or "Place company logo centered at top" (customer-facing)
       - Footer: branding.footer text at bottom
       - For customer-facing: "Professional, clean layout suitable for sharing with homeowners. White background, navy text."
       - For internal: "Data-dense dark theme layout. Navy background with gold/teal accents."

    3. **`buildModuleDataSection(modules: TopicModule[], data: Record<string, unknown>): string`** -- for each module, format: "## ${module.label}\n${JSON.stringify(data[module.id], null, 2)}\nVisual element: ${module.visualElement}" joined by newlines.

    4. **`composePrompt(params: { data: Record<string, unknown>; modules: TopicModule[]; audience: InfographicAudience; scoringResult: ScoringResult; }): string`** -- main entry point. Determines model type from scoringResult and dispatches:
       - If `scoringResult.selectedModel?.webSearchGrounding === true` (NB2): call `composeNB2Prompt`
       - If `scoringResult.selectedModel?.chainable === 'finisher'` (NB Pro direct): call `composeNBProPrompt`
       - If `scoringResult.selectedChain`: call `composeChainPrompts` and return the first step prompt
       - Fallback: compose NB2 prompt

    5. **`composeNB2Prompt(modules, data, branding, audience): string`** -- template:
       ```
       Generate a single-page infographic briefing for a roofing sales representative.

       BRANDING:
       ${brandingInstructions}

       DATA TO VISUALIZE:
       ${moduleDataSection}

       LAYOUT REQUIREMENTS:
       - Single cohesive infographic image, portrait orientation (1024x1792 or similar)
       - ${modules.length} data sections arranged in a clear visual hierarchy
       - Use icons and visual indicators for key metrics
       - All text must be readable at mobile viewing size

       ${webSearchModules.length > 0 ? `WEB SEARCH: Use search_types ["web", "image"] to ground the following modules with live data: ${webSearchModules.map(m => m.label).join(', ')}` : ''}
       ```

    6. **`composeNBProPrompt(modules, data, branding, audience): string`** -- template similar but emphasizes:
       - "COMPOSITION PRIORITY: Create a visually striking, magazine-quality layout"
       - "Use advanced typography hierarchy with clear section delineation"
       - "Emphasize visual polish and professional design over raw data density"
       - No search_types instructions (NB Pro cannot search)

    7. **`composeChainPrompts(params: { modules, data, audience, scoringResult }): { accuracyPrompt: string; refinementPrompt: string }`** -- returns TWO prompts:
       - `accuracyPrompt` (for NB2 step): Focus on data accuracy, include search_types if web modules present. "Focus on accurate data representation. Layout clarity is secondary -- the output will be refined."
       - `refinementPrompt` (for NB Pro step): "Refine the attached reference image into a polished, publication-quality infographic. Maintain all data accuracy from the reference. Improve: typography, spacing, visual hierarchy, color consistency, professional finish." Include branding instructions.

    Export: `composePrompt`, `composeChainPrompts`, `buildBrandingInstructions` (for testing).
  </action>

       - `{ id: 'live-weather', label: 'Live Weather', dataSource: 'getStormHistoryTimeline', requiresWebSearch: true, visualElement: 'Current conditions and forecast panel' }`
       - `{ id: 'neighborhood-context', label: 'Neighborhood Context', dataSource: 'neighborhoodConversionRate', requiresWebSearch: false, visualElement: 'Neighborhood conversion map and stats' }`
       - `{ id: 'insurance-status', label: 'Insurance Status', dataSource: 'insuranceDeadlineCountdown', requiresWebSearch: false, visualElement: 'Claim status timeline with deadline countdown' }`
       - `{ id: 'carrier-intel', label: 'Carrier Intel', dataSource: 'getCustomerOverviewStats', requiresWebSearch: true, visualElement: 'Carrier approval rates and strategy notes' }`
       - `{ id: 'competitor-landscape', label: 'Competitor Landscape', dataSource: 'getCustomerOverviewStats', requiresWebSearch: false, visualElement: 'Competitive positioning comparison grid' }`
       - `{ id: 'company-credentials', label: 'Company Credentials', dataSource: 'getCustomerTitleData', requiresWebSearch: false, visualElement: 'Certifications, warranties, and trust indicators' }`
       - `{ id: 'contact-info', label: 'Contact Info', dataSource: 'getCustomerTitleData', requiresWebSearch: false, visualElement: 'Rep contact card with QR code' }`

       Export as `AVAILABLE_MODULES`.

    4. **`CUSTOMER_FACING_SIGNALS: string[]`** -- language patterns that indicate customer-facing audience: 'send to', 'share with', 'for the homeowner', 'for the customer', 'leave behind', 'leave-behind', 'hand out', 'give to', 'email to customer', 'show the customer'.

    5. **`inferAudience(text: string): InfographicAudience`** -- check if input text contains any CUSTOMER_FACING_SIGNALS (case-insensitive). If match found, return 'customer-facing'. Otherwise return 'internal'.

    6. **`parseIntent(text: string): Promise<IntentParseResult>`** -- main entry point:
       a. Call `inferAudience(text)` for audience determination (fast, local)
       b. Call `getAIRouter().classify()` with:
          - `text`: the user's natural language input
          - `categories`: AVAILABLE_MODULES.map(m => m.id) -- the module IDs
          - `multiLabel: true` -- user may want multiple topics
       c. From ClassifyResult.categories, filter by `confidence >= 0.3`
       d. Map matched category labels back to TopicModule objects from AVAILABLE_MODULES
       e. If no modules matched (all below threshold), return default set: ['property-overview', 'storm-timeline', 'lead-scoring', 'next-steps']
       f. Return `{ modules: matchedModules, audience, confidence: avgConfidence }`

    7. **Error handling:** If AI Router call fails, fall back to keyword matching:
       - Check text for keywords: 'weather' -> live-weather, 'storm' -> storm-timeline, 'insurance' -> insurance-status, 'roof' -> roof-assessment, 'property' -> property-overview, 'competitor' -> competitor-landscape, 'meeting' -> insurance-status + carrier-intel
       - Return matched modules with confidence: 0.5

    Export: `parseIntent`, `IntentParseResult`, `AVAILABLE_MODULES`.
