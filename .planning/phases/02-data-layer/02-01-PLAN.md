  <action>
    Create `src/features/infographic-generator/utils/infographicDataAssembler.ts`:

    1. **Re-export existing deck-generator functions** -- import and re-export getCustomerTitleData, getCustomerOverviewStats, getPropertyIntelData, getStormHistoryTimeline from `@/features/deck-generator/utils/dataAggregator`. These serve as data sources for infographic modules that need the same data.

    2. **Severity weight map** -- define `const SEVERITY_WEIGHTS: Record<string, number> = { minor: 1, moderate: 2, severe: 3, catastrophic: 5 }`.

    3. **`cumulativeStormExposure(customerId: string): Promise<number>`** -- fetch `/api/customers/${customerId}` to get weatherEvents array from the customer API response (same pattern as getStormHistoryTimeline). Sum `SEVERITY_WEIGHTS[event.severity] || 1` for each event. Return the total.

    4. **`daysSinceLastContact(customerId: string): Promise<number>`** -- fetch `/api/customers/${customerId}` to get interactions array. Sort by date descending. Return `Math.floor((Date.now() - new Date(interactions[0].date).getTime()) / 86400000)`. Return -1 if no interactions.

    5. **`neighborhoodConversionRate(customerId: string): Promise<number>`** -- fetch customer to get zipCode, then fetch `/api/canvassing/pins?zipCode=${zipCode}`. Count pins where `outcome === 'closed-won'` vs total pins. Return `Math.round((won / total) * 100)`. Return 0 if no pins.

    6. **`insuranceDeadlineCountdown(customerId: string): Promise<number>`** -- fetch customer to get claims via `/api/customers/${customerId}`. Filter claims with status in ['pending', 'approved']. For each, compute deadline = dateOfLoss + filing window (default 730 days / 2 years, PA = 365 days). Return minimum days remaining. Return -1 if no active claims.

    7. **`DATA_SOURCE_MAP: Record<string, (customerId: string) => Promise<unknown>>`** -- maps dataSource string names to functions: 'getCustomerTitleData' -> getCustomerTitleData, 'getCustomerOverviewStats' -> getCustomerOverviewStats, 'getPropertyIntelData' -> getPropertyIntelData, 'getStormHistoryTimeline' -> getStormHistoryTimeline, 'cumulativeStormExposure' -> cumulativeStormExposure, 'daysSinceLastContact' -> daysSinceLastContact, 'neighborhoodConversionRate' -> neighborhoodConversionRate, 'insuranceDeadlineCountdown' -> insuranceDeadlineCountdown.

    8. **`assembleDataForModules(customerId: string, modules: TopicModule[]): Promise<Record<string, unknown>>`** -- for each module, look up `DATA_SOURCE_MAP[module.dataSource]`. Call all in parallel with `Promise.allSettled`. Return object keyed by `module.id` with each resolved value (or `{ error: reason }` for rejected).

    Follow the same error handling pattern as dataAggregator.ts: try/catch with graceful fallbacks returning default values on failure.
